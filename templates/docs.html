API Documentation

Integrate ChilliDoc AI into your applications with our RESTful API

Getting Started

The ChilliDoc AI API provides simple REST endpoints for plant disease detection. All API requests should be made to: https://api.chillidocai.com

Authentication

Currently, the API is open and does not require authentication. For production use, API keys will be required.

Endpoints

POST /api/predict

Upload a chilli plant image and receive disease prediction with confidence scores and treatment recommendations.

Request

Content-Type: multipart/form-data

Body Parameters:

  • file (required) - Image file (JPG, JPEG, PNG, max 16MB)

Example Request (cURL)

curl -X POST https://api.chillidocai.com/api/predict \ -F "file=@chilli_leaf.jpg"

Example Request (Python)

import requests url = "http://localhost:5000/api/predict" files = {"file": open("chilli_leaf.jpg", "rb")} response = requests.post(url, files=files) result = response.json() print(result)

Example Request (JavaScript)

const formData = new FormData(); formData.append('file', fileInput.files[0]); fetch('http://localhost:5000/api/predict', { method: 'POST', body: formData }) .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error));

GET /api/diseases

Retrieve information about all detectable diseases.

Example Request

curl http://localhost:5000/api/diseases

Response

{ "success": true, "diseases": { "Chilli__Anthacnose": { ... }, "Chilli__Leaf_Curl_Virus": { ... }, ... }, "total_diseases": 5 }

GET /api/disease/:disease_name

Get detailed information about a specific disease.

Example Request

curl http://localhost:5000/api/disease/Chilli__Anthacnose

GET /api/health

Check API health status and model availability.

Response

{ "status": "healthy", "model_loaded": true, "classes_loaded": true, "num_classes": 5, "timestamp": "2025-12-28T10:30:45.123456" }

Error Handling

Error Response Format

{ "success": false, "error": "Error message description" }

HTTP Status Codes

  • 200 - Success
  • 400 - Bad Request (invalid file, missing parameters)
  • 404 - Not Found
  • 500 - Internal Server Error

Rate Limits

Current version has no rate limits. Production API will implement:

  • Free Tier: 100 requests/day
  • Basic: 1,000 requests/day
  • Enterprise: Unlimited (custom pricing)
